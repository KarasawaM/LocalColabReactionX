
## 🖥️ Transition State Optimization and IRC Calculations

`LCRX` supports **transition state (TS) optimization** and subsequent **IRC calculations** using the `Sella` optimizer.  
See the [Sella GitHub repository](https://github.com/zadorlab/sella) for details.

You can preview a template by running:
```sh
lcrx template ts
```

### 📗 Example: TS Optimization and IRC
```toml
[structure]
input = "input.xyz"         # or .pdb, .gjf, .sdf
charge = 0                  # total charge
mult = 1                    # multiplicity (2S+1)

[calculator]
type = "UMA"                # Universal Models for Atoms
model_name = "uma-s-1p1"    # or "uma-m-1p1"
task = "omol"               # omol task for molecules.
device = "gpu"              # or "cpu"

[calculation]
type = "ts"                 # Transition State Search using Sella.
fmax = 0.001                # maximum force for convergence criterion (eV angstrom^-1)
maxstep = 1000              # maximum number of optimization steps
internal = true             # or true. Use internal coordinates. default: true
initial_hess = "vib"        # initial hessian. "vib": ASE Vibrations. "sella": Sella's default estimate.
                            # or "/path/to/hessian.csv | .npy" hessian_2d() format.
update_hess = "vib"         # hessian update method. "vib": ASE Vibrations. "sella": Sella's default update.

[[calculation.irc]]         # Subsequent IRC calculation. Remove this block to skip.
irc_fmax = 0.05             # maximum force for convergence criterion (eV angstrom^-1). default: 0.05
irc_dx = 0.1                # in angstrom sqrt(amu). step size for IRC. default: 0.1
irc_steps = 100             # maximum number of IRC steps. default: 100
endpoint_opt = "LBFGS"      # or "FIRE", "LBFGS", "BFGSLINESEARCH". Optimize the IRC endpoints.
                            # fmax and maxstep settings are taken from the TS opt. false: skip endpoint optimization.
```

#### Hessian Calculation

**Initial Hessian**
The initial Hessian matrix for TS optimization can be specified using `initial_hess`.
Options include:

* `"vib"`: generated by ASE `Vibrations` class
* `"sella"`: Sella’s default guess
* a file path to a Hessian in ASE `hessian_2d()` format (`.csv` or `.npy`)

**Hessian Update**
The Hessian update method can be specified using `update_hess`.
Options are:

* `"vib"`: ASE `Vibrations` class
* `"sella"`: Sella’s default update
